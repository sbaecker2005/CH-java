Usuários (CRUD)

POST /api/usuarios
Cadastrar operador → 201 Created | 400 CPF inválido
GET /api/usuarios
Listar todos → 200 OK
GET /api/usuarios/{id}
Buscar por ID → 200 OK | 404
PUT /api/usuarios/{id}
Atualizar dados → 200 OK
DELETE /api/usuarios/{id}
Remover → 204 No Content (ADMIN)

Autenticação JWT

POST /api/auth/login
Body: {email, senha} → {"token":"eyJ..."}

Como usar o token:

Adicione no header de todas as requisições protegidas:

Authorization: Bearer <token>

Perfis disponíveis:

  • ROLE_ADMIN → acesso total, incluindo DELETE
  • ROLE_OPERADOR → acesso às operações de CRM
